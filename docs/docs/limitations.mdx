# Limitations

## Android only

The module targets Android overlays. It does not provide iOS overlay behavior.

## Expo Go

Expo Go is not supported because the package includes native Android code. Use a development build, custom client, or standalone Android build.

## User-controlled permission

Android requires the user to grant draw-over-apps permission in system settings. Your app can open the settings screen, but it cannot approve the permission for the user.

## Native overlay service

The real floating bubble is hosted by an Android service. Preview components run inside your app and do not create native overlay windows.

## Overlay shared values

Overlay shared values store numbers only. Use them for counters, timer values, selected steps, and small renderer coordination values. Keep objects, private data, and larger app state in your own store. Shared-value keys cross the JavaScript and native boundary, so keep them static and non-sensitive.

## Compose renderer status

`@expo/ui` is optional, and `setComposeBubbleRenderer()` is available. Current Android overlay builds use a React Native fallback for the packaged Compose-flavored renderer to avoid unstable overlay mounting.

## Bubble ID safety

Bubble IDs cross the JavaScript and native boundary. Keep them static and non-sensitive. The module normalizes free-form IDs, but normalization is not a privacy boundary.
