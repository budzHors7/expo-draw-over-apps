# SDK 56 window containers

SDK 56 adds two container components for custom bubble renderers that resize while the overlay is visible.

- `ReactNativeWindowContainer` wraps React Native children in a Reanimated surface.
- `NativeWindowContainer` uses an Expo UI Host/Surface for the Android backdrop when `@expo/ui` is installed, then renders React Native children inside the same clipped window.

Use these names in SDK 56 docs and examples.

## Install

`ReactNativeWindowContainer` uses Reanimated:

```bash
npx expo install react-native-reanimated react-native-worklets
```

`NativeWindowContainer` can use Expo UI for the native backdrop:

```bash
npx expo install @expo/ui
```

`@expo/ui` is optional. Without it, `NativeWindowContainer` falls back to `ReactNativeWindowContainer`.

## React Native container

```tsx
import { Pressable, Text } from 'react-native';
import {
  ReactNativeWindowContainer,
  setOverlaySharedValue,
  useOverlaySharedValueState,
  type BubbleRendererProps,
} from 'expo-draw-over-apps';

const resizeKey = 'window-container-counter';

export function ResizeBubble(_: BubbleRendererProps) {
  const sharedStep = useOverlaySharedValueState(resizeKey);
  const large = sharedStep.value >= 1;

  return (
    <ReactNativeWindowContainer
      width={large ? 238 : 154}
      height={large ? 256 : 172}
      borderRadius={large ? 44 : 28}
      className="border-2 border-rose-400 bg-slate-950 px-4 py-4"
      contentClassName="flex-1 items-center justify-center"
    >
      <Pressable
        onPress={() => setOverlaySharedValue(resizeKey, large ? 0 : 1, 'bubble')}
      >
        <Text style={{ color: 'white', fontWeight: '900' }}>Toggle size</Text>
      </Pressable>
    </ReactNativeWindowContainer>
  );
}
```

## Native container

```tsx
import { Pressable, Text } from 'react-native';
import {
  NativeWindowContainer,
  setOverlaySharedValue,
  useOverlaySharedValueState,
  type BubbleRendererProps,
} from 'expo-draw-over-apps';

const resizeKey = 'window-container-counter';

export function NativeResizeBubble(_: BubbleRendererProps) {
  const sharedStep = useOverlaySharedValueState(resizeKey);
  const large = sharedStep.value >= 1;

  return (
    <NativeWindowContainer
      width={large ? 238 : 154}
      height={large ? 256 : 172}
      borderRadius={large ? 44 : 28}
      seedColor="#0891b2"
      className="border-2 border-cyan-600 px-4 py-4"
      contentClassName="flex-1 items-center justify-center"
    >
      <Pressable
        onPress={() => setOverlaySharedValue(resizeKey, large ? 0 : 1, 'bubble')}
      >
        <Text style={{ color: '#0f172a', fontWeight: '900' }}>Toggle size</Text>
      </Pressable>
    </NativeWindowContainer>
  );
}
```

## Props

Both containers accept:

- React Native `ViewProps`
- `width`, `height`, and `borderRadius`
- `backgroundColor`
- `colorScheme` (`light`, `dark`, or omitted to follow the system)
- `style` and `contentContainerStyle`
- `className` and `contentClassName` for NativeWind apps
- `animationConfig` for Reanimated timing

Use numeric `width`, `height`, and `borderRadius` values when you want smooth transitions. When you omit background props, the containers follow the device light/dark setting; pass `backgroundColor`, a style background, or `colorScheme` when you need a fixed look. Swapping unrelated wrapper layouts can still jump because Android is moving an overlay window outside the normal app tree.

The example resize fixtures use one shared native value, `window-container-counter`, so the React Native and Native Expo UI resize bubbles show the same size step when you switch between them.
