# Code of Conduct

## Our Standard

We want this project to be welcoming, practical, and respectful for everyone.

People participating in this repository are expected to:

- be respectful and constructive
- assume good intent, while still giving honest feedback
- focus on the work and avoid personal attacks
- welcome contributors with different levels of experience
- help keep discussions technical, clear, and collaborative

## Unacceptable Behavior

The following behavior is not acceptable in issues, pull requests, discussions, commit messages, or other project spaces:

- harassment, intimidation, threats, or hate speech
- discriminatory or insulting language
- trolling, baiting, or deliberately disruptive behavior
- sharing private information without permission
- repeated bad-faith arguments or harassment after a boundary has been set
- sexualized language or unwanted advances
- retaliation against anyone who reports a problem

## Scope

This Code of Conduct applies to all project spaces for this repository, including:

- GitHub issues
- pull requests and review comments
- discussions related to the docs site, example app, and module code

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, commits, pull requests, or other contributions that do not follow this Code of Conduct.

Depending on severity, maintainers may:

- ask for a change in behavior
- remove inappropriate content
- lock or close threads
- temporarily or permanently block participation

## Reporting

If you experience or witness behavior that violates this Code of Conduct, contact the maintainer at `haniseanda@gmail.com`.

If the report is not safety-sensitive, you can also open an issue asking for maintainer review, but email is preferred for private concerns.
